# 台股分析系統改進清單

> 以股市新手角度測試後規劃之改進項目。尚未實作。

## 實作流程規範（給 AI 助理）

實作每個項目時嚴格遵守以下步驟：

1. **讀懂需求**：先讀該項目的「問題」與「建議」，確保理解要做什麼
2. **評估變動範圍**：判斷需要修改哪些檔案（前端 HTML/JS、後端 Python、設定檔等）
3. **修改程式碼**：
   - 只改必要部分，不碰無關的程式碼
   - 修改前先讀取要改的檔案，確認當前內容
4. **即時驗證語法**：
   - 修改 HTML/JS 後：檢查大括號 `{}` 與括號 `()` 是否成對平衡，確認無遺漏引號或拼接錯誤
   - 修改 Python 後：`python3 -m py_compile <檔案>` 檢查語法
5. **試跑程式碼確認功能正常**：
   - 啟動 Flask 測試 client：從 `/` 首頁確認渲染正常
   - 呼叫 `/search?q=2330` 確認搜尋正常
   - 呼叫 `/analyze` → `/stream/<task_id>` 確認分析流程可跑完，SSE 正常回傳 `result:` 與 `done:` 事件
   - 若非測試 client 可測的功能（如 UI 互動），在 `templates/index.html` 搜尋對應的 JS 函數確認呼叫路徑完整
6. **確認新增項目有成功補上**：
   - 從使用者角度模擬操作流程，確認修改確實生效
   - 例如：新增 favicon → 確認 `/picture/icon/icon-192.png` 回傳 200 且 Content-Type 正確
   - 例如：新增 API → 用 test client 打 endpoint 確認回傳符合預期格式
7. **標記完成**：
   - 只有在步驟 5、6 都通過後，才能將該項目的 `[ ]` 改為 `[x]`
   - 若驗證失敗，回頭修正程式碼後重新從步驟 4 開始

> 原則：**先驗證再標記完成**，不做無實質驗證的完成宣告。

## P0 — 新手使用核心痛點

### 1. 缺少「整體評分/總評」機制
- [x] **問題**：新手看到一堆數字（PE、PEG、健康度、殖利率等）無法快速判斷「這檔股票到底好不好」。
- [x] **解決**：在 preview 頂端新增「綜合評級」徽章（A/B/C/D），整合健康度（50%）、估值（25%）、成長（25%）加權計算，附顏色條與分數。
- [x] **更動**：`templates/index.html` — 新增 `computeOverallRating()` JS 函數、`ratingArea` div、CSS `.rating-badge`、LANG_MAP 詞條。

### 2. 分析速度過慢（~23.5s）
- [ ] **問題**：從按下「開始分析」到看到結果需約 23 秒，新手可能等不及關閉頁面。
- [ ] **建議**：
  - 將 30 天營收爬蟲與 5 年股利資料改為背景預先快取
  - 顯示「預估剩餘時間」或階段性進度百分比
- [x] **已實作**：視覺化進度條 — 解析 `[N/M]` 步驟編號更新百分比，非編號步驟對應固定權重，完成後顯示 100% 並自動淡出。
- [x] **更動**：`templates/index.html` — 新增 `progressBar`/`progressFill`/`progressText` HTML、SSE log handler 內進度更新邏輯、LANG_MAP 詞條。

### 3. 新手教學未涵蓋指標意義
- [x] **問題**：onboarding 僅說明操作步驟（輸入→分析→預覽→下載），未說明**各指標代表的意義**。
- [x] **解決**：在頁面右上方新增「📖 名詞解釋」按鈕，點開彈窗顯示 GLOSSARY 全部 9 項解釋。
- [x] **更動**：`templates/index.html` — 新增 `glossaryBtn` 按鈕、`glossaryModal` 彈窗、`showGlossary()`/`hideGlossary()`/`renderGlossary()` JS 函數、CSS `.glossary-item`、LANG_MAP 詞條。

### 4. 錯誤訊息不夠友善
- [x] **問題**：當 yfinance 資料源超時或回傳 None，log 顯示「部分資料取得不完全」但新手不知道哪些資料缺失、影響多大。
- [x] **解決**：在 preview 指標區上方新增「資料完整性」狀態列，以彩色 pill 顯示每項資料（股價✅/🔴、營收✅/🔴/➖ETF、EPS✅/🔴/➖ETF、股利✅/🟡、新聞✅/🟡）的可用狀態。
- [x] **更動**：`templates/index.html` — 新增 `dataStatus` div、renderPreview 內 `completeness` 陣列邏輯。

## P0（續）— ETF 與個股區隔化

### 5. ETF 應使用完全不同的評分機制
- [x] **問題**：目前 ETF 和個股共用同一套綜合評級（健康度 50% + 估值 25% + 成長 25%），但 ETF 沒有營收/EPS/本益比，現有評分對 ETF 毫無意義。
- [x] **解決**：新增 `computeETFOverallRating()`，從 yfinance 取得的 ETF 特有資料維度計算：費用率（35%）、折溢價（25%）、流動性（20%）、殖利率（20%）。
- [x] **更動**：`templates/index.html` — 新增 `computeETFOverallRating()` JS 函數；`renderPreview` 內 `data.is_etf` 時自動切換。

### 6. ETF Preview 指標卡片應與個股完全不同
- [x] **問題**：ETF 現在仍顯示「本益比」「營收年增」「PEG」等卡片（資料空缺時才隱藏），新手看到一堆「—」會困惑。
- [x] **解決**：ETF 模式時 gridMain 改顯示：收盤價、NAV、折溢價%（含顏色）、費用率%（含顏色）、AUM（含顏色）、日均量、殖利率、新聞情緒。
- [x] **更動**：`templates/index.html` — `renderPreview` 內 `if (etf)` 分支渲染不同卡片集。

### 7. ETF 隱藏對其無意義的區塊
- [x] **現已處理**：
  - 合理價區間（便宜價/合理價/昂貴價）→ 隱藏（title + container 均 display:none）
  - 營收/EPS 圖表 → 已隱藏 ✓
  - 報酬預估（「到合理價 +X%」）→ 隱藏（title + container 均 display:none）
  - 操作建議 → 改用 ETF 版（溢價/折價建議、費用率建議、配息建議、發行商說明）
  - 檢查清單 → 改用 ETF 版（費用率 ≤ 0.5%、折溢價 ≤ 1%、規模 ≥ 100 億、日均量 ≥ 1000 張、連續配息 ≥ 3 年）
  - 健康度評分 → ETF 模式不顯示，改由 ETF 評級替代
  - PDF 報告 → 營收分析/EPS 分析章節在 ETF 模式下自動跳過，基本資訊欄位替換為 ETF 特有欄位
- [x] **更動**：`templates/index.html` + `report/generator.py`

### 8. ETF 新增專屬區塊
- [x] **ETF 基本資訊卡片**：顯示發行商（fund_family）與類型（category）
- [x] **更動**：`templates/index.html` — 新增 `etfFundCard` 與 `etfFundInfo`；`renderPreview` JS 內 ETF 分支填入資料。
- [ ] **待未來實作**：
  - 同類型 ETF 比較（0050 vs 006208 vs 其它）
  - 折溢價歷史圖表
  - ETF 費用分析詳細說明

### 9. ETF 的資料完整性狀態列應反映 ETF 特有欄位
- [x] **解決**：dataStatus 的 pill 在 ETF 模式改為：`股價 ✅/🔴`、`NAV ✅/🔴`、`費用率 ✅/🟡`、`折溢價 ✅/🟡`、`AUM ✅/🟡`、`股利 ✅/🟡`、`新聞 ✅/🟡`
- [x] **更動**：`templates/index.html` — `renderPreview` 內 `if (etf)` 分支的 `completeness` 陣列邏輯。

## P1 — 高影響力功能

### 5. 搜尋支援產業與關鍵字
- [ ] **問題**：目前搜尋僅依 stock_id 與名稱匹配；新手可能想搜「半導體」看有哪些股票可分析。
- [ ] **建議**：`/search?q=` 後端加入產業名稱與 aliases 模糊比對，前端 autocomplete 顯示產業分類提示。

### 6. 與大盤（0050）比較
- [ ] **問題**：新手看到 2330 本益比 20，但不知道大盤（0050）平均本益比多少，無從判斷高低。
- [ ] **建議**：在估值卡片旁新增「vs 大盤」小標，顯示 0050 同期本益比與殖利率作為參考基準。

### 7. 加入「再次分析」與「重新整理資料」
- [ ] **問題**：目前同一 stock_id 在 5 分鐘內會回傳快取，新手無法手動強制更新資料。
- [ ] **建議**：在結果頁顯示「資料時間戳記」與「🔄 重新整理」按鈕，清除快取後重新分析。

### 8. 下載按鈕顯示檔案資訊
- [ ] **問題**：新手按下「下載完整 PDF」後不知道檔案多大、要等多久。
- [ ] **建議**：在按鈕旁顯示 PDF 大小（如「PDF 2.3 MB」），下載中顯示 spinner。

### 9. 查詢歷史增強
- [ ] **問題**：歷史記錄僅存 stock_id，重新點擊會重新分析（即使快取還在），無法區分已分析過與未分析。
- [ ] **建議**：在歷史標籤上標示「✓ 已分析」狀態，並加入「一鍵比較已分析股票」功能。

## P2 — 中優先級功能

### 10. 夜間模式 / 主題切換
- [ ] **問題**：背景顏色選擇器只能選單色，沒有真正的深色模式。
- [ ] **建議**：加入「🌙 深色模式」切換，儲存至 localStorage，完整改變所有卡片與文字的配色。

### 11. 分享分析結果
- [ ] **問題**：無法將分析結果連結分享給他人。
- [ ] **建議**：在結果區新增「🔗 複製分享連結」按鈕，URL 帶入 stock_id 參數（如 `/?stock=2330`），載入時自動分析。

### 12. 新聞標題預覽
- [ ] **問題**：preview 中只顯示「新聞情緒」數字，看不到實際新聞標題。
- [ ] **建議**：新增可展開的新聞列表（至多 5 則），點擊可查看標題與來源。

### 13. 報酬預估說明不夠直觀
- [ ] **問題**：「到合理價 +X%」的計算方式新手不易理解。
- [ ] **建議**：在報酬預估表格上方加入一行白話文說明（如「若股價回到歷史中位數估值，預期報酬約為…」）。

### 14. 搜尋無結果時提示
- [ ] **問題**：搜尋無結果時 dropdown 直接消失，無任何提示。
- [ ] **建議**：顯示「找不到符合「XXX」的股票」提示文字。

## P3 — 低優先級 / 未來優化

### 15. 貓圖重新整理按鈕
- [ ] **問題**：側邊貓圖每次頁面載入隨機選取，但無法在不重新整理的情況下更換。
- [ ] **建議**：在圖片上加一個小 icon 點擊可重新隨機選圖。

### 16. 載入骨架（Skeleton Screen）
- [ ] **問題**：分析完成後內容一次性出現，畫面跳動。
- [ ] **建議**：在分析過程中逐步顯示灰色 placeholder 卡片，資料到來後逐一填入。

### 17. PDF 預覽功能
- [ ] **問題**：新手需下載 PDF 才能知道報告內容是否值得。
- [ ] **建議**：在「下載 PDF」按鈕旁新增「👁 快速預覽」按鈕，於新分頁顯示前 2 頁。

### 18. 個股所屬產業表現
- [ ] **問題**：只知道個股數據，不知道所屬產業整體趨勢。
- [ ] **建議**：在產業比較卡片旁加入「產業平均本益比」與「產業平均殖利率」參考線。

### 19. 關聯企業 / 上下游顯示
- [ ] **問題**：新手不知道這檔股票的供應鏈或關聯企業。
- [ ] **建議**：在基本資料區加入「所屬集團」或「主要客戶/供應商」資訊（若 yfinance 有提供）。

### 20. 支援更多 ETF 資訊
- [ ] **問題**：ETF 分析僅顯示折溢價建議與追蹤指數，缺少成分股組成與費用率。
- [ ] **建議**：加入 ETF 費用率（expense ratio）、前十大成分股、追蹤誤差等 ETF 特有指標。

### 21. 營收/EPS 資料新鮮度提示
- [ ] **問題**：營收與 EPS 可能來自數月前的財報，新手可能誤以為是最新資料。
- [ ] **建議**：在營收與 EPS 卡片旁顯示資料月份/季度，若超過 2 個月未更新則標示「⚠ 較舊資料」。

### 22. 股利連續配發圖表
- [ ] **問題**：目前只顯示「連續 X 年配息」文字，無視覺化。
- [ ] **建議**：加入簡單的長條圖顯示近 5 年股利金額變化。

### 23. 支援股票代號直接從 URL 參數分析
- [ ] **問題**：無法透過網址直接觸發分析（如 `/?q=2330`）。
- [ ] **建議**：前端在頁面載入時檢查 URL query string，若有 `?q=` 則自動填入並開始分析。

### 24. hover 到名詞時顯示浮動解釋卡
- [ ] **問題**：GLOSSARY 存在但新手不知道可以 hover ⓘ 圖示。
- [ ] **建議**：在首次載入時短暫顯示「🛈 將滑鼠移到各指標可查看說明」提示條，3 秒後自動消失。
